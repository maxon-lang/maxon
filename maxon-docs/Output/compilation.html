Ir Generation

The compiler generates LLVM Intermediate Representation (IR) code.

Optimization Modes

Optimized Mode (default):

  • Maximum performance
  • Smaller code size
  • Harder to debug

Debug Mode (-g flag):

  • Preserves source structure
  • Easier debugging
  • Slower execution

Example

Source code:

function main() returns int
    var sum = 0
    var i = 0
    while i < 10 'loop'
        sum = sum + i
        i = i + 1
    end 'loop'
    return sum
end 'main'
Expected Output
Exit Code: 45

Optimized IR (simplified):

define i32 @main() {
  ret i32 45  ; Loop fully computed at compile time
}

Debug IR would preserve the full loop structure with all variables and operations.
