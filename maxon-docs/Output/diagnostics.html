Missing Return Error

Functions that declare a return type must return a value on all code paths.

Error Example

function test() returns int
    // Error: No return statement
end 'test'

Error message:

Semantic Error: Function 'test' must return a value of type 'int'
Note: All execution paths through the function must end with a return statement

Solution

Ensure every path returns:

function test(x int) returns int
    if x > 0 'check'
        return 1
    end 'check'
    return 0  // Handles else case
end 'test'

Unknown Keyword Error

Using an undefined keyword or bare identifier as a statement causes a parse error.

Error Example

function main() returns int
    var x = 0
    foo
        x = 5
    end 'foo'
    return x
end 'main'
Error Output
In file 'temp_fragment.maxon':
Unexpected identifier 'foo'
  Note: Did you forget an assignment (=), function call (), or keyword?
  Location: line 4, column 5

Common Solutions

When you see this error, consider:

  • Function call: foo() instead of foo
  • Assignment: foo = 5 instead of foo
  • Check spelling of keywords: while, if, var, let, return, break

Unused Parameters

Maxon requires all function parameters to be used. Declaring unused parameters causes a compilation error.

Example Error

function add(a int, b int) returns int
    return a  // Error: 'b' is unused
end 'add'

Error message:

Semantic Error: The parameter 'b' is declared but its value is never used

Solution

Only declare parameters you need:

function identity(a int) returns int
    return a  // OK: 'a' is used
end 'identity'
