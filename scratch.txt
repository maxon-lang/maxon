4. Iterator Invalidation
This is a specific type of logic error common in languages like C++ or Java.

The Problem: You are iterating through a list, and inside the loop, you add or remove an element. This often causes the internal pointer of the iterator to point to garbage or skip elements.

Rust's Solution: While you have an active iterator (which "borrows" the list), the borrow checker prevents you from modifying the list. The code simply won't compile if you try to push an item into a collection you are currently looping over.




3. Data Races
A data race occurs in multi-threaded programming when two or more threads access the same memory location concurrently, and at least one of them is writing.

The Problem: Because the timing of threads is unpredictable, the final value of the memory becomes non-deterministic, leading to "Heisenbugs" that are nearly impossible to debug.

Rust's Solution: The borrow checker enforces the Aliasing XOR Mutation rule: You can have many "readers" (immutable references) OR exactly one "writer" (mutable reference), but never both at the same time.
